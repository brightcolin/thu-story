# 游戏养成系统触发与功能说明 v2.2

---

## 一、触发条件总览

| 触发方式 | 涉及接口 | 触发的养成逻辑 |
|----------|----------|---------------|
| 玩家执行活动 | `POST /activities/execute` | 精力/健康/科研/社工变化 → 解锁检测 → 归零检测 → 学期切换GPA结算 |
| 玩家上课 | `POST /class/attend` | 课程掌握度变化 → 精力消耗 → 归零检测 |
| 玩家与NPC对话 | `POST /chat` | NPC好感度变化 → 解锁检测（恋爱线等） |
| 宵禁晚归 | `POST /player/penalties/curfew` | 精力-30/健康-20 → 归零检测 |
| 未按时吃饭 | `POST /player/penalties/meals` | 每漏一餐精力-15/健康-10 → 归零检测 |
| 食堂吃饭 | `POST /activities/execute(eat_canteen)` | 自动标记当前餐段已吃，阻止后续缺餐扣罚 |
| 加入社工组织 | `POST /social/join` | 设置社工组织 → 解锁加成效果 |
| 社工晋升 | `POST /social/promote` | 检查社工能力阈值 → 职位晋升 |
| 时间自然流逝 | 后端实时时钟 | 游戏时间推进（现实0.9秒=游戏1分钟） |
| 学期边界跨越 | 活动导致 `semester_index` 变化 | 自动：冻结课程→计算GPA→清空课表 |
| 手动学期交割 | `POST /semester/transition` | 同上 |

## 二、核心属性与成长公式

### 2.1 基础属性

| 属性 | 范围 | 初始值 | 增长来源 | 消耗来源 |
|------|------|--------|----------|----------|
| 精力(energy) | 0-100 | 60 | 吃饭+20, 休息+30, 睡觉+50 | 上课-10, 自习-15, 做实验-20 |
| 健康(health) | 0-100 | 60 | 吃饭+15, 运动+20, 睡觉+20 | 缺餐-10/餐, 晚归-20 |
| 科研(research) | 0-100 | 0 | 做实验+2, 导师面谈+3 | - |
| 社工(social) | 0-100 | 0 | 社团活动+3, 组会+5 | - |
| GPA | 0.0-4.0 | 0.0 | 学期末按掌握度加权计算 | 缺勤降低掌握度 |

### 2.2 课程掌握度 → GPA 映射

每门课有独立掌握度（0-100），学期末按以下规则映射为单科绩点：

| 掌握度 | 绩点 |
|--------|------|
| ≥95 | 4.0 |
| ≥90 | 3.7 |
| ≥85 | 3.3 |
| ≥80 | 3.0 |
| ≥75 | 2.7 |
| ≥70 | 2.3 |
| ≥65 | 2.0 |
| ≥60 | 1.5 |
| <60 | 0.0（不及格） |

**累计GPA = Σ(单科绩点 × 学分) / Σ学分**

### 2.3 掌握度变化规则

| 操作 | 掌握度变化 | 说明 |
|------|-----------|------|
| 准时上课 | +5 | 科协加成×1.1 |
| 迟到上课 | +2 | 迟到15分钟以上 |
| 缺勤 | -10 | 课已结束仍未到 |
| 图书馆自习（选科目） | +3 | 科协加成×1.1 |
| 老师答疑 | 由NPC对话触发 | 间接提升 |

### 2.4 社工组织加成

| 组织 | 加成 |
|------|------|
| 学生会 | NPC好感度提升量×1.1 |
| 团委 | 社工能力提升量×1.15 |
| 科协 | 课程掌握度提升量×1.1 |

## 三、状态流转

### 3.1 精力/健康生命周期

```
正常(>30) ──活动消耗──→ 警告(1~30) ──继续消耗──→ 归零(0)
                                                    │
                                   ┌────────────────┘
                                   ▼
                           精力归零: 跳过1天, 恢复到50/50
                           健康归零: 跳过3天, 恢复到50/50
                                   │
                                   ▼
                              正常(50) ──→ 继续游戏
```

**关键规则**：
- **边沿检测**：只有从 >0 变为 0 时触发（已经为0不重复触发）
- **健康优先**：同时归零只执行生病分支（+3330分钟），不叠加疲惫（+1110）
- **恢复后两项都设为50**，防止立即再次归零的死循环

### 3.2 好感度 → 解锁

```
NPC好感度 0──────→30("友善")──→60("亲密")──→80("知己")──→100
                    │              │              │
                    ▼              ▼              ▼
              张锟霖≥30      沈星辞≥60      林晚晴≥80
              +社工≥20       解锁恋爱线     解锁导师亲密
              →解锁社团      (boyfriend)    (mentor_close)
              (club_joined)
```

### 3.3 学期生命周期

```
学期开始 → 选课(POST /courses/select) → 上课/自习/考试 → 学期结束
                                                           │
                                              ┌────────────┘
                                              ▼
                                    semester_transition()
                                    ├── 冻结课程(mastery_final快照)
                                    ├── 计算不及格学分
                                    ├── 重算累计GPA
                                    ├── 写入 gpa_committed
                                    ├── 清空旧课表
                                    └── 下一学期开始(再次选课)
```

### 3.4 结局分支

```
                        ┌─ GPA≥3.7 + 科研≥70 → 保研直博
                        ├─ GPA≥3.8 + 科研≥80 + 社工≥40 → 出国留学
大四下学期 → 结局判定 ──├─ 社工≥80 + GPA≥3.5 + 科研≥50 → 自主创业
                        ├─ GPA≥3.3 + 科研≥40 → 外校保研
                        ├─ GPA≥2.5 → 考研成功
                        ├─ 始终可选 → 灵活就业
                        └─ 不及格学分≥20 → 退学（强制）
```

## 四、各接口对养成状态的具体影响

### 4.1 `POST /activities/execute`

| 活动 | 精力 | 健康 | 科研 | 社工 | 特殊效果 |
|------|------|------|------|------|----------|
| 图书馆自习 | -15 | - | - | - | 可选科目+3掌握度 |
| 做实验 | -20 | - | +2 | - | 需解锁lab_access |
| 食堂吃饭 | +20 | +15 | - | - | 自动标记当前餐段 |
| 休息 | +30 | +10 | - | - | - |
| 睡觉 | +50 | +20 | - | - | 跳到第二天6:30 |
| 运动 | -5 | +20 | - | - | - |
| 社团活动 | -10 | - | +1 | +3 | 需解锁club_joined |
| 组会 | -10 | - | - | +5 | 需解锁social_org_joined |
| 导师面谈 | -10 | - | +3 | - | - |

带 `flags:["curfew_penalty"]` 时额外：精力-30，健康-20

**每次执行后依次检查**：
1. 精力/健康归零边沿 → 触发faint/hospital
2. NPC好感度变化 → 自动解锁（恋爱线/社团/实验室等）
3. 学期边界跨越 → 学期交割（GPA结算+课表清空）

### 4.2 `POST /class/attend`

| 出勤状态 | 判定条件 | 掌握度 | 精力 |
|----------|---------|--------|------|
| on_time | 到达时间 ≤ 上课+15分钟 | +5 | -10 |
| late | 到达时间 > 上课+15分钟但 < 下课 | +2 | -10 |
| absent | 到达时间 ≥ 下课 | -10 | 0 |

### 4.3 `POST /player/penalties/curfew`

- 精力 -30，健康 -20（在当前值上减）
- 同一游戏日幂等（不重复扣）
- 扣后检查归零

### 4.4 `POST /player/penalties/meals`

- 每漏一餐：精力 -15，健康 -10
- 可同时漏多餐（如早+午 = 精力-30，健康-20）
- 按 (day_block, meal_type) 幂等
- `eat_canteen` 执行时自动标记当前餐段，阻止后续扣罚

### 4.5 `POST /chat`

- NPC回复中 `friendship_change`（-5~+5）应用到好感度
- 学生会成员：正向好感度×1.1
- 好感度变化后自动检查解锁条件

### 4.6 `POST /semester/transition`

- 冻结当前学期所有课程（`mastery_final` = 当前掌握度）
- 按掌握度→绩点映射计算各科成绩
- 累计不及格学分（绩点=0的课程学分之和）
- 重算加权累计GPA并写入 `gpa_committed`
- 清空当前学期课表（`course_schedule`）
- 交割后 `player.gpa` 固定为新值，学期中不再变化

## 五、配置参数汇总

以下参数均在 `database.py` 顶部定义，可通过修改代码调整：

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `REAL_SECONDS_PER_GAME_MINUTE` | 0.9 | 现实9秒=游戏10分钟 |
| `MINUTES_PER_DAY` | 1110 | 每游戏日分钟数(6:30→1:00) |
| `DAYS_PER_SEMESTER` | 28 | 每学期天数 |
| `TOTAL_SEMESTERS` | 8 | 总学期数(大一上→大四下) |
| `MEAL_DEADLINES.breakfast` | 210 | 早餐截止偏移(=10:00) |
| `MEAL_DEADLINES.lunch` | 450 | 午餐截止偏移(=14:00) |
| `MEAL_DEADLINES.dinner` | 870 | 晚餐截止偏移(=21:00) |
| `MEAL_MISS_PENALTY` | {energy:-15, health:-10} | 每漏一餐扣罚 |
| `CURFEW_PENALTY` | {energy:-30, health:-20} | 晚归扣罚 |
| `ENERGY_ZERO_SKIP_DAYS` | 1 | 精力归零跳过天数 |
| `HEALTH_ZERO_SKIP_DAYS` | 3 | 健康归零跳过天数 |
| `RECOVERY_ENERGY_AFTER_SKIP` | 50 | 跳过后精力恢复值 |
| `RECOVERY_HEALTH_AFTER_SKIP` | 50 | 跳过后健康恢复值 |
